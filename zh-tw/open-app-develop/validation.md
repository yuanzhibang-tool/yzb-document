### 所有通過`js`的參數均需要進行驗證,以免傳入非法的參數問題造成異常錯誤

!>分為兩類參數,一類是通用的參數驗證,另外一類是針對每個方法的參數驗證,例如,`tid`為通用驗證參數

### 通用規則寫於此,其他的則寫在對應的`json`傳參示例中參數中.

| 通用參數值 | 規則                                | 說明 |
| ---------- | ----------------------------------- | ---- |
| `tid`      | `require,int,max_length:11,require` |      |
| `app_id`   | `require,int,max_length:11`         |      |
| `name`     | `min:1,max:64`                      |      |
